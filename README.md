## Core Repositories & Relevance to RSI

### [master-key](https://github.com/gsovereing/master-key)
Production-grade Python implementation (FastAPI, PostgreSQL, tree-sitter AST, binary FSRS) of the Whole-First Learning method with strict architectural invariants ("незыблемость gates", authoring/runtime path separation).  
**RSI relevance:** Provides verifiable safety gates and epistemic validation layers that can prevent undetected self-modification and alignment erosion during recursive training loops.

### [llm-associative-blindness-benchmark](https://github.com/gsovereing/llm-associative-blindness-benchmark)
Documents cross-model High-Weight Token Priming — models anchor reasoning in prior irrelevant context despite correct surface answer.  
**RSI relevance:** Critical for detecting data poisoning vectors and hidden associative attacks in self-improvement processes.

### [llm-context-robustness-eval](https://github.com/gsovereing/llm-context-robustness-eval) & [llm-context-inertia](https://github.com/gsovereing/llm-context-inertia)
Automated evaluation frameworks measuring Valence Carryover, Style Inertia, and Context Reset Speed under dynamic perturbations.  
**RSI relevance:** Enables detection of cumulative alignment drift in long-horizon agentic and recursive interactions.

### [claude-conversation / findings](https://github.com/gsovereing/claude-conversation)
Identification of Latent Conversational Dominance (F-001) with full reproducible transcripts.  
**RSI relevance:** Highlights risks of models subtly dominating or steering their own improvement trajectories.
